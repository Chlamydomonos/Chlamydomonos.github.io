Cloudomonos去中心化系统开发笔记（一）
Cloudomonos

发布时间: 2022-01-09, 02:16:42

上次更新: 2022-01-10, 11:56:56

最近，我正在考虑开发一个去中心化Minecraft服务器系统。但是，看了这个B站视频之后，我萌生了一个想法：为什么不把这个系统加入一个更加通用的新系统中呢？于是，我建立了这个项目——Cloudomonos去中心化系统。现在，这个项目还是一个很初步的想法。它的功能将会涵盖同步文件、控制单片机设备、转发Minecraft服务器数据包等实用功能，也可能会加入像上述视频中系统一样的分布式计算功能。接下来的内容，就会是Cloudomonos系统的完整搭建过程。

一. 基于树莓派的网桥

现在，我首先会搭建这个系统的基础——利用树莓派组成的网桥。由于我并没有公网IP，要想让不同位置的设备互相访问，就只能使用内网穿透或者虚拟局域网服务。而我恰好有一块树莓派，可以利用它，同时搭建内网穿透和虚拟局域网，并且组成网桥，使得我能远程访问家里的局域网。

2. 树莓派上Zerotier的安装

Zerotier是一个高效并且免费的虚拟局域网工具，非常适合用于Cloudomonos的搭建。要使用Zerotier，需要先到其官网注册一个账号，再创建一个网络。这里，我已经拥有了如图所示的网络：

接下来，为了保证Zerotier网络分配的IP地址不与路由器分配的IP地址冲突，对网络的IP自动分配池进行设置：

由于路由器分配的地址从192.168.0.100开始，这里的地址改用192.168.1.100开始。

我的电脑已经用Windows版的Zerotier客户端连接到了网络，并自动分配了地址192.168.1.189。

接下来，需要把树莓派也连接到Zerotier网络。首先利用官网上给出的命令在树莓派上安装Zerotier客户端：

curl -s https://install.zerotier.com | sudo bash

客户端安装完成后，可以使用sudo zerotier-cli -h命令查看客户端的使用帮助。这里，直接使用sudo zerotier-cli join 8286ac0e471d5a3c加入Zerotier网络。

加入Zerotier网络后，为了方便后续操作，我给树莓派分配了一个不在自动分配池中的特殊IP地址——192.168.6.1。

为了验证树莓派是否加入了Zerotier网络，可以用已经加入网络的电脑试图通过192.168.6.1连接树莓派。

3. 树莓派上网桥的建立

Zerotier官网上提供了用树莓派桥接物理局域网与Zerotier网络的教程。我曾在学校按照教程配置过树莓派，但效果却十分玄学。因此，这次配置，我决定按照教程，从头开始，再尝试解决出现的问题。

由于Zerotier官网上的教程是英文的，这里给出一个简化版——

  • 开始配置树莓派前，确认你有办法在不联网的情况下操作树莓派。（这至少需要键盘和显示器。由于我只能在家里找到没有USB接口的古老键盘和没有HDMI接口的古老显示器，最终的解决方案是使用电视当显示器并借了一个键盘）

  • 找一根网线，把树莓派与路由器连接。

  • 开始配置前，再次确认一些信息：

    • 你的Zerotier网络ID
    • 你的路由器DHCP分配的IP范围
    • 你的路由器的IP地址

    在我操作时，我的Zerotier网络ID为8286ac0e471d5a3c，路由器DHCP分配的IP范围为192.168.0.100到192.168.0.149，路由器的IP地址为192.168.0.1。同时，我的网络里还有一台IP地址为192.168.0.2的路由器，但这并不影响网络的配置。

  • 首先，更新树莓派系统并重启。输入以下命令：

    sudo apt update && sudo apt -y full-upgrade && sudo reboot

  • 接下来，如果没有安装Zerotier和加入网络，按以下内容操作；否则，略过此步：

    • curl -s https://install.zerotier.com | sudo bash
    • sudo zerotier-cli join 你的Zerotier网络ID

  • 接下来，设置一些系统变量。依次输入以下命令：

    • NETWORK_ID=你的Zerotier网络ID
    • BR_IF="br0"
    • BR_ADDR=你希望网桥在的IP地址（这里最好与路由器的DHCP范围处于同一网段）
    • GW_ADDR=路由器的IP地址

    这里，我把网桥地址设置为了192.168.0.50。

  • 接下来，依次输入以下命令：

    • sudo zerotier-cli set $NETWORK_ID allowManaged=0
    • sudo zerotier-cli listnetworks

    输入这两条命令后，命令行应该会显示类似下图的内容。

    记住其中形如zt********的值（我这里是ztrta3cvd3）。

  • 接下来，设置下一个系统变量：

    • ZT_IF=刚才记住的值

  • 接下来，进入Zerotier网络的配置网站，对网络中的树莓派授权，分配一个IP。（最好与路由器的DHCP在同一网段。我分配了192.168.0.25。）在设置选项中启用全部两个选项。

  • 接下来，卸载树莓派默认的网络系统，启用systemd系统，并配置网桥。按顺序输入以下命令：

    • sudo apt remove --purge --auto-remove dhcpcd5 fake-hwclock ifupdown isc-dhcp-client isc-dhcp-common openresolv
    • sudo ln -sf /run/systemd/resolve/resolv.conf /etc/resolv.conf;
    • sudo systemctl enable systemd-networkd;
    • sudo systemctl enable systemd-resolved;
    • sudo systemctl enable systemd-timesyncd;

    • sudo zerotier-cli set $NETWORK_ID allowManaged=0（这里很奇怪，我也不知道为什么这条要再输一次。似乎不输也是可以的）

    • cat << EOF | sudo tee /etc/systemd/network/25-bridge-br0.network
[Match]
Name=$BR_IF

[Network]
Address=$BR_ADDR
Gateway=$GW_ADDR
DNS=1.1.1.1 # 这是谷歌的DNS服务器，已经被墙。建议换成一个国内可用的地址。
EOF
      cat << EOF | sudo tee /etc/systemd/network/br0.netdev
[NetDev]
Name=$BR_IF
Kind=bridge
EOF
      cat << EOF | sudo tee /etc/systemd/network/25-bridge-br0-zt.network
[Match]
Name=$ZT_IF
      [Network]
Bridge=$BR_IF
EOF
      cat << EOF | sudo tee /etc/systemd/network/25-bridge-br0-en.network
[Match]
Name=eth0 # 也许是en*，建议先用ifconfig确认一下

[Network]
Bridge=$BR_IF
EOF

  • 接下来，对Zerotier网络进行设置。

    • 把IP自动分配池设为与路由器的DHCP在同一网段并互不重叠。

    • 在Managed Routes一栏删除默认路由，并添加一个新路由，左侧填入一个可以包含路由器DHCP池的CIDR地址，右侧填入你的路由器IP地址。例如，我的路由器分配网段是192.168.0.x，所以左侧填入192.168.0.0/16。

  • 最后，使用sudo reboot重启树莓派，理论上来说网桥就配置成功了。

我按照教程配置了树莓派。然而，最后重启之后，树莓派却根本没有连上有线网络。根据漫长的检查，我发现了问题——我在启用了systemd之后才连接网线，树莓派的自动网络配置系统此时已经被卸载，不能正常连接到网络。

于是，我只能给树莓派重新刷系统，再来一遍。

这次，我注意了以下事项：

  • 最重要的一点：在开始任何配置前，先把树莓派连上有线网络！
  • 此外，连上有线网络后，在路由器中给树莓派分配一个固定IP。

再次输完所有指令并重启树莓派后，树莓派确实连上了网络，路由器中也出现了一个地址为网桥地址的匿名设备。

这时，从树莓派的IP地址已经不能SSH连接到树莓派了，取而代之的是网桥地址。对于VNC，也需要从网桥地址连接。

然而，这时，树莓派出现了下一个问题：不能解析域名。这明显是DNS服务器的问题，因此我开始尝试重新指定DNS服务器。

我参考了多篇文章，尝试了多个方案，最终发现，这篇文章中提到的方法是有效的。当然，树莓派并未默认安装vim，要想编辑文件，只能用vi或tee等其他方法。

最终，在修改了DNS地址后，网桥的配置完全成功。Zerotier网络和真实的局域网被完全统一为了一个网络。两个网络中的计算机可以随意互相访问，基本没有任何障碍。
